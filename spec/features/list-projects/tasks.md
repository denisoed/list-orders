# Orders

**Context:** Реализовать страницу «Мои проекты» с карточками, плавающей кнопкой добавления и баннером пустого состояния на основе предоставленного HTML-шаблона.

## Main directions

### UI

- [ ] Создать страницу `/projects` (или актуальный маршрут) и перенести структуру HTML в Vue-компонент Nuxt.
- [ ] Разбить макет на компоненты: шапка страницы, список проектов, карточка проекта, пустой баннер, плавающая кнопка.
- [ ] Настроить Tailwind классы и Material Symbols, обеспечить корректное отображение в светлой и тёмной темах.
- [ ] Добавить прогресс-бары с вычисляемым процентом ширины.

### State & Data

- [ ] Определить интерфейс Project и подготовить моковые данные или подключить источник данных при наличии.
- [ ] Добавить условный рендеринг для списка и пустого состояния, покрыть кейс отсутствия проектов.

### Navigation & Accessibility

- [ ] Настроить навигацию карточек (ссылки или `navigateTo`) и обработчик кнопки добавления проекта (роутер/событие-заглушка).
- [ ] Добавить aria-атрибуты для плавающей кнопки и убедиться в доступности основных элементов.

## Supporting orders

- [ ] Документация: описать новую страницу и использование компонентов в README или разделе фронтенда.
- [ ] Observability: Not required — reason: UI без бэкенд-взаимодействия.
- [ ] Code review and PR: подготовить изменения к ревью и описать ключевые моменты в PR.

## Definition of Done

- [ ] Все задачи выполнены и протестированы на реальном интерфейсе (desktop + mobile режимы браузера).
- [ ] Актуальные unit/e2e/integration тесты пройдены или добавлены при необходимости.
- [ ] Документация и операционные инструкции обновлены.
- [ ] `/spec/core/verify.md` выполнен после завершения всех задач для контроля чеклиста.
