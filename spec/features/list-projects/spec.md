# Список проектов

**Specification:** Создать страницу «Мои проекты» с Tailwind-разметкой из предоставленного HTML: добавить в приложение новый экран со списком карточек проектов, прогресс-барами и кнопкой добавления, а также отобразить баннер-заглушку, если проекты отсутствуют.

## User Stories

- Как пользователь, я хочу видеть список моих проектов в удобном виде с названием, прогрессом и статусом задач, чтобы быстро оценивать состояние работы.
- Как пользователь, я хочу иметь заметную кнопку добавления проекта, чтобы сразу перейти к созданию нового проекта.
- Как пользователь, я хочу получать понятный баннер с подсказкой, если проектов ещё нет, чтобы понимать, что делать дальше.

## Main scenarios and rules

- Страница доступна из меню (точка входа определяется при реализации) и отображает заголовок «Мои проекты» с иконкой поиска.
- При наличии проектов выводятся карточки из HTML-шаблона: иконка папки, название, текст с прогрессом и линейный индикатор выполнения.
- Каждая карточка кликабельна и ведёт на страницу проекта (ссылка-заглушка `#` до подключения реальной навигации).
- Если список проектов пуст, выводится баннер-заглушка с текстом «Пока проектов нет, создайте новый» (точное оформление нужно определить при интеграции).
- Плавающая кнопка добавления проекта (`+`) закреплена в правом нижнем углу и доступна в светлой и тёмной темах.
- Нужно предусмотреть переключение темы (light/dark) как в шаблоне: подхватываются классы Tailwind `dark:`.
- Код страницы должен использовать существующий стек (Nuxt 3 + Tailwind) и аккуратно интегрировать HTML с компонентным подходом.

## Non-functional requirements

- Страница должна корректно отображаться на ширине от мобильных экранов (360px) до десктопов, сохраняя адаптивность Tailwind.
- Совместимость с текущей конфигурацией Tailwind и шрифтом Inter, без подключения лишних ресурсов.
- Обеспечить базовую доступность: aria-лейблы для кнопки добавления, читабельные тексты, достаточная контрастность (учитывая dark/light).
- Код должен быть компонентным, легко тестируемым, с разделением данных (список проектов) и представления.

## Assumptions

- Источник данных о проектах пока отсутствует — предполагается временный мок или загрузка из стора позже.
- Логика поиска пока не реализована, кнопка отображается как декоративный элемент до подключения функционала.
- Точная навигация по карточкам будет задана позже; сейчас достаточно ссылки-заглушки или привязки к маршруту `/projects/{id}` при появлении данных.
- Баннер пустого состояния должен быть реализован в соответствии с дизайн-системой проекта; если готового компонента нет, можно добавить локальный.
