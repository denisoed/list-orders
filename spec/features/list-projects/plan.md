# Implementation Plan

**Plan:** Интегрировать страницу «Мои проекты» в Nuxt-приложение с использованием предоставленного HTML и Tailwind-классов, разбив макет на переиспользуемые компоненты и добавив обработку пустого состояния.

## Data sources / schemas

- Использовать временный статический массив объектов проектов (`id`, `name`, `completedOrders`, `totalOrders`) до подключения API.
- Предусмотреть преобразование данных в вычисляемые поля: процент выполнения, подписи «X из Y задач выполнено».
- Описать интерфейс Project в TypeScript для типизации компонента списка.
- Хранить моковые данные в композиции или локальном состоянии страницы; в будущем заменить на запрос к реальному API/Pinia store.

## Contracts and interfaces

- Определить компонент `ProjectCard` с пропсами: `title`, `completedOrders`, `totalOrders`, `href`.
- Ввести контракт для пустого состояния (`hasProjects`) и отображение баннера с текстом «Пока проектов нет, создайте новый».
- Кнопка добавления проекта должна иметь aria-label «Добавить проект» и вызывать событие/роутер-переход (пока заглушка).
- Обновить маршрут `/projects` (или аналогичный) в Nuxt pages, обеспечить SSR совместимость и экспорт default компонента страницы.

## Architecture / Components

- Создать страницу в `app/pages/projects/index.vue` (уточнить текущую структуру) с разметкой на основе HTML-шаблона.
- Разбить шаблон на компоненты: шапка, список карточек, карточка проекта, баннер пустого состояния, плавающая кнопка.
- Настроить Tailwind классы в соответствии с шаблоном, подключить шрифт Inter через глобальные стили (уже есть в проекте).
- Добавить условный рендеринг: если массив проектов пуст, показывать баннер; иначе — список карточек.
- Обеспечить реактивное вычисление ширины прогресс-бара через inline-style с процентом.
- Перенести Material Symbols через глобальное подключение или компонент Icon, чтобы не дублировать `<link>` на странице.

## Risks

- Несоответствие Tailwind классов существующей дизайн-системе: требуется проверка с UI/UX.
- Возможные конфликты с глобальными стилями или отсутствием подключенного шрифта/иконок — нужно убедиться в наличии зависимостей.
- Плавающая кнопка может перекрывать другие элементы на малых экранах; важно проверить отступы и адаптив.
- Отсутствие реальных данных может привести к расхождению интерфейса при интеграции с API: необходимо предусмотреть обработку загрузки/ошибок в будущем.

## Assumptions

- В проекте уже настроен Tailwind с поддержкой `dark:` классов, дополнительных правок не потребуется.
- Роут `/projects` свободен и может использоваться для новой страницы.
- Подключение Material Symbols глобально доступно; при необходимости будет добавлено в layout, чтобы страница выглядела как в макете.
- Контент баннера пустого состояния будет согласован с продакт-менеджером; текущая формулировка удовлетворяет требованиям.
