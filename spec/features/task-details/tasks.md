# Tasks

**Context:** нужно создать новую динамическую страницу деталей заказа. Она будет открываться при нажатии на карточку заказа.

## Main directions
### API и данные
- [ ] Уточнить контракт API деталей заказа и добавить/обновить функцию `fetchOrderDetails` в `data/orders` или соответствующем composable.
- [ ] Обновить мок-данные/фикстуры для локального стенда, обеспечив наличие позиций, истории и вложений.
- [ ] Добавить/актуализировать типы данных заказа и связанных сущностей.

### UI и маршрутизация
- [ ] Создать динамическую страницу `pages/orders/[id].vue` с загрузкой данных через `useAsyncData` и обработкой состояний загрузки/ошибки.
- [ ] Реализовать секции страницы согласно дизайн-системе (хедер, статусы, основные сведения, товары, контакты, описание, вложения, история, действия), обеспечив адаптивность и поддержку тёмной темы.
- [ ] Настроить переход из карточки заказа в списке на новую страницу и кнопку возврата обратно.

### Взаимодействия и состояния
- [ ] Добавить обработчики для пользовательских действий (например, смена статуса/маркировка завершения) с временными заглушками и обновлением UI.
- [ ] Реализовать кэширование/повторную загрузку данных и обновление страницы после изменения статуса.
- [ ] Покрыть возможные пустые состояния и ошибки информативными сообщениями.

## Supporting tasks
- [ ] Documentation: update relevant instructions и wiki с описанием новой страницы и маршрута.
- [ ] Observability: убедиться, что ошибки загрузки логируются, и при необходимости добавить пользовательские события/метрики.
- [ ] Code review and PR: подготовить изменения для ревью и описать ключевые сценарии в описании PR.

## Definition of Done
- [ ] All tasks are completed and tested.
- [ ] Relevant unit/e2e/integration tests pass successfully.
- [ ] Documentation and operational instructions are updated.
- [ ] `/spec/core/verify.md` is executed after completing all tasks to verify the task list.
