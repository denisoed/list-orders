# Task Details Page

**Specification:** нужно создать новую динамическую страницу деталей заказа. Она будет открываться при нажатии на карточку заказа.

## User Stories
- Как менеджер по поддержке, я хочу открывать страницу с деталями заказа, нажимая на карточку в списке, чтобы быстро получить всю информацию о заказе без дополнительной навигации.
- Как менеджер склада, я хочу видеть статус и состав заказа на странице деталей, чтобы оперативно подготовить его к отгрузке.
- Как руководитель, я хочу видеть историю действий и обновлений заказа, чтобы контролировать исполнение SLA.

## Main scenarios and rules
- Страница доступна по динамическому маршруту `/orders/[id]` и подставляет идентификатор заказа из списка.
- При загрузке выполняется запрос данных заказа, включая базовые сведения (номер, клиент, дата, сумма, статус), список товаров, историю изменений, вложения и действия.
- На странице отображаются блоки: заголовок с кнопкой возврата, основные сведения, статусы/бейджи, список товаров, контакт клиента, секция истории, вложения и возможные действия (например, смена статуса).
- Если данные ещё загружаются, показывается скелетон/индикатор. При ошибке загрузки отображается понятное сообщение и кнопка повторить попытку.
- Для отсутствующих данных блоки скрываются или показывают маркеры "Нет данных".
- Требуется поддержка светлой и тёмной тем, адаптивность для планшета/desktop, доступность (ARIA для кнопок, контраст текста).

## Non-functional requirements
- Ответ на запрос деталей должен приходить менее чем за 1 секунду при нормальной нагрузке; кэшировать результат в кэше клиента до обновления статуса.
- Страница должна корректно работать при отключённом JavaScript SSR-фолбэк (Nuxt) и на мобильных устройствах шириной от 360px.
- Использовать существующие токены Tailwind/стили, без внедрения инлайн-стилей вне дизайн-системы.
- Предусмотреть обработку ошибок сети и показ сообщений с логированием в консоль только для отладки.
- Обновление статусов должно происходить без перезагрузки страницы (реактивные данные через composables/store).

## Assumptions
- API для получения деталей заказа уже существует в `data/orders` или может быть добавлено аналогично существующим методам; формат нужно уточнить.
- Доступны ссылки на вложения в виде URL; скачивание происходит по прямой ссылке.
- История изменений приходит отсортированной по дате и содержит акторов и описание действий.
- Требуется уточнение набора доступных действий (кнопок) и прав пользователей перед реализацией.
