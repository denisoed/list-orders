# Tasks

**Context:** Реализовать динамическую страницу публичного профиля коллеги с использованием общих стилей приложения, обеспечив отображение контактной информации и задач с фильтрами и историей.

## Main directions

### UI и навигация

- [ ] Создать динамический маршрут профиля коллеги (`pages/colleagues/[id].vue`) с подключением общего layout и кнопки возврата.
- [ ] Реализовать компонент заголовка профиля с аватаром, именем, ролью и действиями, используя токены цвета и типографики приложения.
- [ ] Добавить блок контактной информации с адаптивной сеткой и ссылками `tel:`, `mailto:`, `tg:`; предусмотреть отображение пустых состояний.
- [ ] Сверстать секцию активных задач с фильтрами-чипсами и карточками задач, поддерживающими светлую/тёмную тему.
- [ ] Отобразить историю задач (выполненные/архивные) с уменьшенной насыщенностью и индикаторами статуса.

### Интеграция данных

- [ ] Настроить загрузку данных профиля через composable (`useAsyncData`/`useFetch`) с обработкой загрузки, ошибок и кешированием.
- [ ] Обновить/добавить типы для профиля, контактов и задач, учитывая опциональные поля и статусы.
- [ ] Подключить локализацию текстов и статусов задач (при наличии i18n).
- [ ] Реализовать отображение Skeleton/Loading state для основных секций до получения данных.

### Тестирование и качество

- [ ] Написать компонентные тесты для ключевых блоков (заголовок, контакты, список задач) и сценариев фильтрации.
- [ ] Добавить e2e/интеграционный сценарий перехода со страницы коллег на профиль и проверку отображения данных.
- [ ] Обновить фикстуры/моки данных задач для покрытия разных статусов и пустых состояний.

## Supporting tasks

- [ ] Документация: описать структуру страницы профиля и используемые компоненты в README или wiki проекта.
- [ ] Observability: убедиться, что ошибки загрузки логируются стандартным способом и при необходимости добавить трекинг событий просмотра профиля.
- [ ] Code review and PR: подготовить описание изменений, скриншоты светлой/тёмной тем и инструкции по проверке.

## Definition of Done

- [ ] Все задачи выполнены и покрыты ручной проверкой ключевых сценариев.
- [ ] Все релевантные unit/e2e/integration тесты успешно проходят.
- [ ] Документация и инструкции обновлены, включены скриншоты страницы.
- [ ] `/spec/core/verify.md` выполнен после завершения всех задач для проверки чек-листа.
